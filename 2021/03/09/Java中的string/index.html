字符串排序

出去经典的快速排序外，根据字符串的特性，我们得出三个全新的字符串排序方式

在介绍前，我们做出如下约定

符号 代表含义
$ W $ 最长字符串长度
$R$ 字符串的基数
$N$ 字符串的数量

三种排序方法均有类似的思维，即依照每一位的不同来排序。这些方法中，低位、高位优先排序方法根据键值排序，而第三种三向字符串排序的方法则结合了键值排序和compareTo排序的特点。其中低位高位排序方法需要大量的辅助空间，而第三种不会，而第一种第二种不涉及大量的远距离交换，因此不会第1，2种排序方式时稳定的，而第三种排序方式则是不稳定的。

1.低位优先排序法

该方法不需要使用递归，类似于数据结构中的基数排序，其思想是从低到高，按位排序，因此在处理短字符串的排序问题时十分高效，且实现起来较为方便，他的时间复杂度为$NW$。

2.高位优先排序法

该方法需要使用递归，其基本思想是在每次排序中将字符串集$S$拆分成$R$个字符串集，之后再对这$R$个字符串集递归调用排序函数，该方法从最高位向下拆分，其时间复杂度从$N$到$NW$不等，但是由此会出现一个问题，由于每次拆分的过程中需要一个辅助数组，因此对于一个较大的字符串集，在拆分的过程中会需要数量极为庞大的空间，比如对于一个15个元素长为10的子字符串集，可能会出现如下情况，他们前九个字符都相等，只有最后一个字符不相等，这时使用该方法会消耗极大的空间，而时间开销也会增加，因此一个处理方法可以是当一个子字符集的长度小于一定值时，将排序方式改为插入排序这样的简单排序方式。该方法较使用于随机字符串的排序。而当遇到有很长前缀相同的时候，由于不断的递归而使用了大量的辅助数组，在基数$R$不太大时这笔开销并不是很显著，而在基数很大的时候由于辅助数组的长度和基数正比，因此会有很大一笔内存开销因此在这样的环境下并不适合使用该方法。

3.三向字符串快速排序

这种方法结合了前两种排序方法与经典三向快速排序方法的特性，即按位比较但不使用辅助数组。这种方法在每一次调用时，根据当前字符集的某一个特定的位将字符集拆分为三个部分，这个拆分的过程使用的是快速排序中的交换排序法，因此这种方法是不稳定的排序方法，但这种方法能够破除基数的限制且契合于字符串的特性，其时间复杂度为$N$到$NW$之间，是一种比较通用的字符串排序方法。
